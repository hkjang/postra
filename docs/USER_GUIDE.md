# Postra 이메일 통합 관리 플랫폼 사용자 가이드

본 가이드는 Postra 이메일 서비스 사용자를 위한 설명서로, 메일 수발신 계정 등록부터 대화형 스레드 조회, 자동화 수신 규칙 적용, AI 의미론적 검색(Semantic Search) 활용법까지 시스템 이용에 필요한 전 과정을 기술합니다.

---

## 1. 시스템 소개 및 개요

**Postra**는 기업 및 조직의 메일 자산을 안전하게 보호하고 통합 관리하기 위해 개발된 차세대 온프레미스 이메일 클라이언트/플랫폼입니다.

### 주요 핵심 특징
- **통합 메일함 지원**: 외부 및 사내 여러 메일 계정(POP3, IMAP, SMTP)을 하나의 안전한 인터페이스에서 통합 관리합니다.
- **제로 트러스트 봉투 암호화(Envelope Encryption)**: 수신된 메일 원문과 첨부파일, 사용자 인증 비밀번호는 최고 수준의 KEK/DEK 암호화로 보호됩니다.
- **AI 의미론적 메일 검색**: 단순 키워드 검색을 넘어 자연어 맥락을 이해하는 AI 인공지능 기반 메일 검색을 제공합니다.
- **스마트 수신 규칙 엔진**: 특정 조건(발신자, 제목, 키워드)에 맞춰 라벨 자동 지정, 알림, 아카이브, 스노우즈(Snooze) 처리를 지원합니다.
- **실시간 악성 첨부 검사**: 수신되는 첨부파일을 자동으로 스캔하여 위험 요소를 사전에 차단합니다.

## 2. 화면 구성 및 인터페이스 (UI) Layout

Postra 웹 인터페이스(`/ui/`)는 효율적인 이메일 관리를 위해 직관적인 3단 반응형 레이아웃으로 설계되었습니다.

### 2.1 주요 UI 영역 안내
1. **사이드바 (Navigation Sidebar)**
   - **새 메일 작성**: 이메일 작성 창 호출 버튼
   - **메일함 목록**: 전체 메일함, 받은 편지함, 안읽은 메일함, 중요 메일함, 보관함(Archive), 휴지통
   - **계정 목록**: 등록된 POP3/IMAP 수신 계정 및 상태 표시
   - **규칙 관리**: 자동화 수신 규칙 설정 메뉴
   - **AI 메일 검색**: 의미론적 벡터 검색 전용 페이지
2. **메일 스레드 목록 (Thread List)**
   - 동일한 대화 주제(References / Subject)로 묶인 이메일 스레드 목록 출력
   - 읽음/안읽음 상태, 라벨, 중요 표시, 첨부파일 유무, 수신 일시 표시
   - 상단 검색바: 실시간 키워드 및 필터 검색
3. **메일 본문 뷰어 (Message Reader)**
   - 선택된 이메일 스레드의 전체 대화 내역 연속 표시
   - Bluemonday 정책 기반으로 스크립트 및 외부 악성 리소스가 안전하게 제거된 HTML Safe 뷰어 제공
   - 답장, 전체 답장, 전달, 아카이브, 삭제, 라벨 추가/제거 액션 버튼

## 3. 메일 계정 연동 및 설정 (Account Setup)

Postra를 이용하기 위해서는 메일을 수신하고 발신할 이메일 계정을 연동해야 합니다.

### 3.1 POP3 / IMAP 수신 계정 등록
1. 웹 인터페이스 좌측 사이드바 하단의 **[계정 관리]** → **[새 계정 추가]** 메뉴를 선택합니다.
2. 다음 항목을 입력합니다:
   - **계정 이름 (Name)**: 사용자 식별용 계정 별칭 (예: `개인 업무 메일`)
   - **이메일 주소 (Email)**: 사용자의 이메일 주소 (예: `user@example.com`)
   - **수신 프로토콜**: `POP3` 또는 `IMAP` 선택
   - **수신 서버 호스트 (Host)**: POP3/IMAP 서버 주소 (예: `pop3.example.com`)
   - **포트 (Port)**: POP3S(995), POP3(110), IMAPS(993), IMAP(143)
   - **보안 설정 (Security)**: `SSL/TLS`, `STARTTLS`, `Plain` (SSL/TLS 권장)
   - **사용자명 (Username)**: 서버 로그인 계정명
   - **비밀번호 (Password)**: 수신 서버 인증 암호
3. 입력 후 **[연결 진단 (Test Connection)]** 버튼을 클릭하여 서버 통신 및 인증을 확인합니다.

### 3.2 발신 SMTP 계정 등록
1. 동일한 계정 설정 화면 하단의 **[발신(SMTP) 설정]** 영역을 작성합니다:
   - **SMTP 호스트**: 발신 서버 주소 (예: `smtp.example.com`)
   - **SMTP 포트**: SMTPS(465), Submission(587), Plain(25)
   - **보안 프로토콜**: `SSL/TLS`, `STARTTLS`, `Plain`
   - **인증 방식 (Auth)**: `PLAIN`, `LOGIN`, `NONE`
   - **SMTP 비밀번호**: 수신 비밀번호와 다를 경우 별도 입력
2. **[저장하기]** 버튼을 누르면 연동 비밀번호가 암호화 SecretStore에 안전하게 격리 저장됩니다.

## 4. 메일 동기화 및 수신 관리 (Mail Sync & Management)

### 4.1 수동 동기화 및 자동 수집
- **수동 동기화 (Manual Sync)**: 계정 목록 옆의 **[동기화 (Sync)]** 버튼을 클릭하면 서버에서 신규 메일을 즉시 수집합니다.
- **자동 동기화 (Auto Sync)**: 관리자 설정에 따라 지정된 주기(예: 5분 단위)마다 백그라운드 스케줄러가 신규 메일을 자동 수집합니다.

### 4.2 이메일 관리 기능
- **라벨 지정 (Labels)**: 메일에 분류 태그(예: `업무`, `긴급`, `프로젝트A`)를 부여하여 체계적으로 정리할 수 있습니다.
- **중요 메일 (Important)**: 별표 아이콘을 클릭하여 중요 메일로 등록하고 중요 메일함에서 모아볼 수 있습니다.
- **스노우즈 (Snooze)**: 지정한 시간(예: 1시간 후, 내일 아침)까지 메일을 메일함에서 잠시 숨겼다가 해당 시각에 다시 상단에 알림 표시합니다.
- **보관 (Archive)**: 처리 완료된 메일을 받은 편지함에서 아카이브함으로 이동하여 메일함을 정돈합니다.

## 5. AI 의미론적 메일 검색 (AI & Semantic Search)

Postra는 단어 일치 검색뿐만 아니라 질문이나 문맥으로 필요한 이메일을 찾아내는 AI 검색을 지원합니다.

### 5.1 검색 방법
1. 좌측 메뉴의 **[AI 메일 검색]**을 클릭합니다.
2. 검색창에 구체적인 질의를 자연어로 입력합니다.
   - *예시*: `"지난달 마케팅 예산 관련 승인받은 보고서 찾아줘"`
   - *예시*: `"서버 장애 대응 관련 보안 조치 사항 이메일"`
3. AI 벡터 엔진이 메일의 본문 및 제목 문맥을 비교하여 연관도가 높은 순서대로 결과를 정렬하여 보여줍니다.

## 6. 이메일 작성 및 발송 (Compose & Send)

### 6.1 이메일 작성 및 초안 저장
1. **[새 메일 작성]** 버튼을 누릅니다.
2. 수신자(`To`), 참조(`Cc`), 제목 및 본문을 작성합니다.
3. 작성 중인 메일은 실시간으로 **초안(Draft)**에 자동 저장되어 작업 손실을 방지합니다.

### 6.2 첨부파일 및 보안 스캔
- 파일 첨부 시 사내 악성코드 및 바이러스 스캔 엔진(ClamAV/YARA)이 자동으로 안전성 검사를 수행합니다.
- 의심스러운 첨부파일(예: 실행 파일 `.exe`, 이중 확장자, 압축 폭탄)은 자동으로 차단되어 안내 메시지가 표시됩니다.

### 6.3 발송 및 재시도 (Outbox Retry)
- **[발송]** 버튼을 클릭하면 즉시 SMTP 서버를 통해 메일이 발송됩니다.
- 네트워크 일시 장애로 발송 실패 시 아웃박스(Outbox) 재시도 큐로 이동하며, 백그라운드 워커가 자동으로 재발송을 시도합니다.

## 7. 자주 묻는 질문 (FAQ)

Q1. 메일 동기화 중 오류 메시지가 뜹니다. 어떻게 해야 하나요?
> 계정 설정의 [연결 진단]을 실행하여 비밀번호 입력 오류(`secret_acquire` 실패)인지, 네트워크 접속 타임아웃인지 먼저 확인하세요. 비밀번호가 변경된 경우 계정 수정 페이지에서 암호를 다시 입력 후 저장하시면 자동 복구됩니다.

Q2. 첨부파일 다운로드가 차단되었습니다.
> Postra는 사내 보안 정책에 따라 위험 확장자나 악성 코드가 포함된 첨부파일을 격리 조치합니다. 필요한 파일인 경우 시스템 관리자에게 보안 정책 완화를 요청하십시오.
